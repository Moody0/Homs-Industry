# إعداد صناعة حمص

## متغيرات البيئة
انسخ `.env.example` إلى `.env.local` وضع القيم الحقيقية:

```bash
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=sb_publishable_your_key_here
SUPABASE_SERVICE_ROLE_KEY=
NEXT_PUBLIC_SITE_URL=http://localhost:3000
GOOGLE_MAPS_API_KEY=
```

`NEXT_PUBLIC_SUPABASE_URL` و`NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` مطلوبان لتشغيل الموقع. `SUPABASE_SERVICE_ROLE_KEY` اختياري ويجب أن يبقى server-only؛ لا تستخدمه داخل أي Client Component ولا تضعه تحت اسم يبدأ بـ `NEXT_PUBLIC_`.

## Auth بالبريد الإلكتروني
المشروع يستخدم Supabase Auth لتخزين كلمات المرور وإدارة الجلسات. تسجيل الدخول يدعم البريد الإلكتروني أو اسم المستخدم، لكن اسم المستخدم ليس مزود Auth أصلياً في Supabase؛ لذلك يتم حله server-side عبر دالة `resolve_login_email` ثم تسجيل الدخول بالبريد وكلمة المرور.

قبل الإنتاج:
- فعّل Email Auth من لوحة Supabase، واضبط إعدادات تأكيد البريد حسب بيئة التطوير أو الإنتاج.
- رقم الهاتف يبقى بيانات تواصل داخل `profiles` وليس مزود تسجيل دخول.
- أبقِ رسائل فشل الدخول عامة لتقليل كشف الحسابات.
- يوجد rate limit بسيط داخل السيرفر لتقليل محاولات الدخول المتكررة؛ في الإنتاج الكبير يفضّل استبداله بمخزن دائم مثل Redis أو Upstash.

## إنشاء أول مدير
لا يوجد تسجيل عام للمدراء. أنشئ حساباً عادياً أولاً، ثم اجعله مديراً بإحدى الطريقتين:

```sql
update public.profiles
set role = 'admin'
where id = 'REGISTERED_USER_UUID';
```

أو افتح جدول `profiles` من Supabase Dashboard وعدّل `role` للمستخدم المطلوب إلى `admin`.

## بيانات تجريبية كاملة
يوجد ملف seed للتطوير في `supabase/seed.sql`. يشمل حسابات تجريبية، محلات مقبولة ومعلقة ومرفوضة، صور محلية، ساعات عمل، منتجات بأسعار، تقييمات، مفضلة، وإعلانات نشطة/منتهية/معطلة.

حساب المدير التجريبي:

```txt
email: admin@example.com
username: admin
password: admin123
```

ملاحظة مهمة: هذا الحساب مخصص للتطوير فقط. لا تشغّل `supabase/seed.sql` في بيئة إنتاجية كما هو.

## الصور و Supabase Storage
الباكتات الحالية عامة للقراءة حتى يمكن عرض الصور مباشرة. هذا يعني أن أي صورة مرفوعة قد تكون قابلة للوصول عبر رابطها حتى لو كان النشاط `pending`، لذلك:
- لا تعرض روابط صور الأعمال غير الموافق عليها إلا للمالك أو المدير.
- نظّم ملفات الأعمال تحت `user_id/business_id/...`.
- تحقّق من نوع وحجم الملفات قبل الرفع.
- الأنواع المقبولة حالياً: JPG وPNG وWebP.
- الحد الحالي: 5MB لصور الأعمال والإعلانات، و3MB لصور الفئات.

## الخرائط
صفحة النشاط تستخدم iframe مبنياً على `latitude` و`longitude` ولا تحتاج مفتاح Google Maps. ميزات متقدمة مثل Places API أو رسم مسارات أو Autocomplete ستحتاج `GOOGLE_MAPS_API_KEY` وتنفيذاً منفصلاً.

## الإعلانات المدفوعة
لا يوجد نظام فوترة داخل التطبيق حالياً. الإدارة تنشئ الإعلانات وتحدد التواريخ والأولوية يدوياً من لوحة الإدارة. أي دفع إلكتروني أو فواتير يحتاج مرحلة مستقلة.

## الصور الإنتاجية
لقطة الشاشة هي مرجع الشكل العام فقط وليست مجموعة أصول إنتاجية. استخدم صوراً مرفوعة من Supabase أو صور seed/fallback الصناعية الموجودة في `public/images`.

## مفتوح الآن وقريب مني
لا تعرض حالة “مفتوح الآن” إلا إذا كانت بيانات ساعات العمل موجودة فعلياً ومتصلة بالنشاط. ميزة “قريب مني” تعتمد على سماح المتصفح بالموقع الجغرافي ووجود إحداثيات صحيحة للنشاطات.
